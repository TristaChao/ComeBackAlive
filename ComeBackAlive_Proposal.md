# 遊戲企劃提案 Proposal

**專案代號**：《活著回來就好》

**類型**: 潛行生存 × 料理模擬 × 高風險取捨
單人遊戲 / 氣氛導向 / 系統驅動

---

## 一、專案概述（Executive Summary）

《活著回來就好》是一款以弱小角色生存體驗為核心的遊戲。
玩家扮演一隻剛滿一個月大的鼠鼠，在充滿天敵的環境中覓食、潛行、生存，並將冒險所得帶回鼠窩料理。

本作不以戰鬥或變強為主軸，而是以：
- 聲音與視野限制
- 資源風險管理
- 料理作為唯一成長手段

來塑造高度緊張、但溫暖的遊戲體驗。

核心精神只有一句話：
> 每天，活著回來就好。

---

## 二、核心特色（Key Pillars）

1.  **高風險潛行探索**
    - 玩家幾乎無反擊能力
    - 依賴：
        - 潛行
        - 聲音管理
        - 視野判斷
    - 每一次行動都可能引來死亡

2.  **料理即成長**
    - 沒有傳統升級系統
    - 料理 = 回復 + Buff + 成長
    - 複雜料理帶來更高收益，但也更高操作壓力

3.  **物理限制取代數值限制**
    - 沒有「一次只能煮 3 道菜」這類硬性規則
    - 料理上限由：
        - 廚具數量
        - 櫃台數量
        - 食材實際存量
    - 玩家能「看見」自己的極限

---

## 三、遊戲核心循環（Core Gameplay Loop）

1.  鼠窩中配置廚房、備料
2.  外出探索地圖
3.  潛行避開敵人
4.  撿拾 / 挖掘 / 奪取物資
5.  遭遇敵人 → 逃跑 / 引誘 / 放棄
6.  活著回到鼠窩
7.  進入料理模式
8.  烹飪 → 食用 → 獲得 Buff
9.  擴建鼠窩 → 下一輪探索

---

## 四、鼠窩與料理系統（Cooking Hub）

### 4.1 鼠窩定位
- **功能定位**：安全區 + 壓力料理場
- **視覺與操作邏輯**：接近《速速上菜》

### 4.2 初始廚房配置
- 切台 ×1
- 烤台 ×1
- 櫃台 ×1
- 盤架 ×1（4 個盤子）
- 倉庫 ×1（容量有限）

### 4.3 家具配置系統
- **一般模式下**：
    - 按 E 拿起 / 放下家具
    - 玩家可自由調整動線與效率

---

## 五、料理系統設計

### 5.1 料理模式
- 按 Y 進入 / 離開料理模式
- **料理模式中**：
    - 無法移動家具
    - 專注操作料理流程

### 5.2 料理流程（範例）
1.  從倉庫取出食材 → 放櫃台
2.  切台切菜（長按 E，進度條）
3.  烤台烤熟（進度條）
4.  盤架取盤 → 承盤
5.  食用

### 5.3 料理失敗設計（已確認）
- 料理失敗不會直接消失
- 會變成「燒焦的食物」
    - 可食用
    - 回復量極低
    - 可能附帶負面效果
- **懲罰來源**：浪費玩家冒險得來的資源

---

## 六、探索地圖內容設計

### 6.1 地圖物件類型
1.  **散落物（低風險）**
    - 少量食物 / 建材
2.  **埋藏物（中風險）**
    - 需挖掘（進度條）
    - 持續製造動靜
    - 可中斷，進度保留
3.  **動物窩點（高風險高報酬）**
    - 大量資源
    - 敵人密集看守
    - 鼓勵引誘與策略行動

---

## 七、潛行、聲音與敵人系統

### 7.1 視野限制
- 玩家只能看到前方錐形區域
- 背後完全不可見
- 需要主動觀察

### 7.2 聲音系統
- 所有行為會產生「動靜」
- 敵人依聲音與距離反應

### 7.3 心跳音效機制（已確認）
- **條件**：
    - 敵人進入危險距離
    - 尚未被發現
- 敵人越近 → 心跳越急促
- 潛行狀態下心跳聲較弱
- 被發現後停止播放（避免干擾）
- 👉 作為非 UI 的玩家提示與氣氛工具

---

## 八、引誘與氣味系統

玩家可丟出：
- 垃圾
- 料理
- 食材

不同物品：
- 吸引不同敵人
- 吸引範圍不同

**核心策略**：用食物換安全路線

---

## 九、角色數值系統（Player Stats）

### 已定義數值
- 血量
- 飽食度
- 跑速
- 各動作動靜範圍
- 頰囊大小
- 挖掘速度

### 成長方式
- 僅透過料理 Buff
- 死亡後 Buff 全部清空
- 家園與食譜永久保留

---

## 十、Buff 與成長系統（Buff & Progression）

本作不使用傳統等級或技能樹系統。
角色成長完全依賴料理所帶來的 Buff。

Buff 分為兩種類型：短期 Buff 與 長期 Buff，並且具有明確的風險與失去條件。

### 10.1 短期 Buff（Temporary Buffs）
- **定義**
    - 由「一般料理」或「即時食用料理」提供
    - 用於單次外出探索
    - 具備時間限制
    - 死亡或時間結束即消失
- **設計目的**
    - 鼓勵玩家為「這一趟外出」做準備
    - 增加出門前的策略選擇
    - 不會造成永久強化，避免滾雪球

**短期 Buff 範例**
| Buff 類型 | 效果 |
| :--- | :--- |
| 奔跑加速 | 跑速小幅提升 |
| 潛行專注 | 行動動靜範圍降低 |
| 聽覺敏感 | 敵人聲音提示距離增加 |
| 耐餓 | 飽食度下降速度降低 |
| 冷靜 | 心跳音效觸發距離縮短 |

### 10.2 長期 Buff（Persistent Buffs）
- **定義**
    - 由「複雜料理」或「高階料理」提供
    - 代表鼠鼠的長期生存能力提升
    - 在存活期間持續生效
    - 角色死亡後全部清空
- **設計目的**
    - 提供中長期目標
    - 獎勵成功生存多天的玩家
    - 同時保留死亡的風險重量

**長期 Buff 範例**
| Buff 類型 | 效果 |
| :--- | :--- |
| 頰囊擴張 | 頰囊容量增加 |
| 消化強化 | 最大飽食度上限提升 |
| 熟練廚師 | 特定料理失敗容錯增加 |
| 穩定挖掘 | 挖掘速度提升 |
| 家務熟練 | 家具互動速度提升 |

### 10.3 Buff 與料理關係
- **料理複雜度越高**：
    - 提供 Buff 越強
    - 越有可能提供「長期 Buff」
- **燒焦食物**：
    - 只提供極低回復
    - 不提供任何 Buff
    - 可能附帶短期負面效果

### 10.4 Buff 疊加與限制（設計原則）
- ⚠️ **尚需設計細則，但原則已定義**
- 同類型 Buff 不疊加，只取最高效果
- 不同類型 Buff 可同時存在
- 長期 Buff 數量存在上限（避免失控）
- Buff UI 儘量低存在感（圖示 + 描述）

#### （補充）對專案價值的說明（給主管）
> 這套 Buff 系統的價值在於：
> - 取代傳統升級系統
> - 降低數值膨脹風險
> - 讓「料理」成為所有系統的核心
> - 讓死亡仍然有重量，但不摧毀長期進度

#### 一句設計確認（很重要）
> 短期 Buff 決定你「這一趟敢不敢賭」，
> 長期 Buff 決定你「已經活了多久」。

---

## 十一、死亡與重生

- **死亡條件**：
    - 被敵人捕獲
    - 飽食度歸零
- **重生於鼠窩**
- **清空**：
    - 頰囊
    - 所有 Buff
- **保留**：
    - 廚具
    - 鼠窩大小
    - 食譜知識

---

## 十二、專案風險與對策（給主管看的）

| 風險 | 對策 |
| :--- | :--- |
| 新手壓力過高 | 燒焦食物機制緩衝 |
| 系統複雜 | 表格化設計、逐步解鎖 |
| 玩家囤料 | 食材久放可能產生氣味 |

## 十四、結語

《活著回來就好》不是一款追求爽快感的遊戲，
而是一款讓玩家在壓力中做選擇，在選擇中活下來的作品。

它的記憶點不在勝利，而在——
每一次回家的那一刻。
